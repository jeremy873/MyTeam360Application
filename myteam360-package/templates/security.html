Threat-by-Threat Comparison

Every OpenClaw vulnerability. How we handle it.

Threat OpenClaw MyTeam360
🌐 Exposed Ports
OpenClaw bound to 0.0.0.0:18789 by default — tens of thousands of instances were exposed to the entire internet.
✗ Wide Open by Default
Default config listened on all network interfaces. Tens of thousands of live instances found by Bitsight in healthcare, finance, and government networks.
✓ Zero Exposed Ports
Binds to 127.0.0.1 only. Production traffic routes through Cloudflare Tunnel — server IP hidden, no public ports open. Optional Tailscale VPN for private-only access.
🔑 Credential Theft
API keys, OAuth tokens, and passwords stored in plaintext config files. Malicious skills exfiltrated them via curl to attacker servers.
✗ Plaintext Credentials
Keys stored in plaintext YAML/JSON config. 820+ malicious ClawHub skills specifically targeted credential files. One campaign stole crypto wallets from 335 skills.
✓ Encrypted at Rest
All API keys encrypted with AES-256 in the database. DLP scanner blocks accidental exposure of keys in chat. No third-party plugin system — no supply chain attack vector.
🐚 Shell Execution
OpenClaw runs terminal commands, browses the web, and modifies files with full system access. A compromised agent = a compromised machine.
✗ Full System Access
Agents can execute arbitrary shell commands, read/write any file, browse the web, and access all connected services. Microsoft recommended isolated VMs only.
✓ No System Access
Agents are chat-only. They cannot execute commands, access files, browse the web, or touch anything outside the conversation. The AI responds with text — it cannot act on your machine.
🧩 Malicious Plugins
ClawHub marketplace had 820+ malicious "skills" including crypto stealers, reverse shells, and credential harvesters — found within minutes of looking.
✗ Open Marketplace
Community-submitted skills run as trusted code. A single threat actor published 386 malicious packages. Security researcher found malware within 2 minutes of browsing.
✓ No Plugin System
There is no marketplace, no user-submitted code, no skill system. Your 18 agents are built-in with fixed capabilities. Nothing external runs on your server.
💉 Prompt Injection
Malicious instructions embedded in emails, documents, and web pages could hijack the agent's actions — including sending data to attackers.
✗ Actions from Injections
Because OpenClaw can act (run commands, send emails, modify files), a prompt injection = real-world damage. Attackers embedded instructions in Google Docs that auto-executed.
✓ No Actions to Hijack
Even if a prompt injection succeeds, the agent can only produce text. It cannot execute commands, send emails, modify files, or access external services. The blast radius is zero.
🔓 No Authentication
No rate limits on login. No brute force protection. No MFA. WebSocket connections from any website could hijack the agent silently.
✗ Minimal Auth
No brute force protection. No rate limits on passwords. CVSS 8.8 vulnerability let any website silently take full control via localhost WebSocket. CVE-2026-25253.
✓ Full Auth Stack
Rate limiting, account lockout after failed attempts, MFA/TOTP, session management, API token scoping, IP allowlisting, Google OAuth. Every endpoint is authenticated.
📡 Data Exfiltration
Skills could silently send data to external servers. Network calls happened without user awareness. Persistent memory files could be poisoned for delayed attacks.
✗ Silent Exfiltration
Skills explicitly instructed the agent to curl data to external servers. Silent execution with no user awareness. Memory files (SOUL.md) poisoned for persistent backdoors.
✓ DLP + No Outbound
DLP scanner blocks SSNs, credit cards, and API keys from leaving chat. Agents have no internet access — they cannot make network calls. Conversations encrypted in the database.
🔍 Network Discovery
OpenClaw broadcast its presence via mDNS, exposing filesystem paths, SSH availability, and host details to anyone on the local network.
✗ mDNS Broadcasting
Broadcasts hostname, filesystem paths, CLI binary location, and SSH port to the local network. Makes reconnaissance trivial for attackers.
✓ Invisible
No mDNS, no broadcasting, no local discovery. With Cloudflare Tunnel, even the server IP is unknown. With Tailscale, only your authorized devices know it exists.
📋 Audit Trail
Organizations couldn't see what OpenClaw was doing. No logging of actions, no audit trail, no way to detect compromise.
✗ No Audit Logging
No built-in audit trail. CrowdStrike and Microsoft both flagged the inability to monitor agent actions as a critical enterprise risk.
✓ Complete Audit Log
Every login, message, API call, and admin action logged with timestamp, IP, user ID, and action type. Exportable. Filterable by severity.
