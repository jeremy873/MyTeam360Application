Security Principles

01Bring Your Own Key (BYOK)

MyTeam360 does not hold AI provider API keys. You bring your own key from Anthropic, OpenAI, Google, xAI, or open-source providers. Your key is stored encrypted in our database and used only when you make an AI request. We never see your conversations, never train on your data, and never pay for your AI usage.

Your key, your controlYou can rotate, revoke, or change your API key at any time from Settings. Changes take effect immediately.
Provider independence10 providers, 28 models. If one shuts down, switch in one click. Run locally with Ollama for zero cloud dependency.

02Encryption

All sensitive data — API keys, tokens, and credentials — is encrypted using AES-256-GCM before storage. The encryption key is stored as an environment variable on the server, separate from the database. Even if the database were compromised, encrypted fields are unreadable without the master key.

Algorithm: AES-256-GCM
Nonce: 12 bytes, randomly generated per encryption
Key storage: Environment variable (MT360_ENCRYPTION_KEY)
Transport: TLS 1.2+ (HTTPS enforced)

03Platform Key Governance

On the rare occasion the platform itself needs to make an AI API call (system maintenance, update notifications), the call is governed by a strict policy:

Owner approval requiredEvery platform-internal use case must be explicitly approved by the platform owner before any API call can be made.
Daily limits enforcedEach approved use case has daily call limits and cost caps that cannot be exceeded.
Full audit trailEvery platform key call is logged: what was called, when, which model, how many tokens, estimated cost.
Hardcoded forbidden listUser conversations, content generation, CRM operations, and any user-facing feature are permanently forbidden from using platform keys. This is enforced in code, not configuration.

04Safe Integration Policy

MyTeam360 connects to external services (Google Calendar, Outlook, email, flights, weather) using a strict read-only policy we call the OpenClaw Rule:

READ: ✓ Always allowed
DISPLAY: ✓ Always allowed
SUGGEST: ✓ Always allowed
CREATE: ⚠ Only with explicit user confirmation + audit log
MODIFY: ✗ Never
DELETE: ✗ Never
MOVE: ✗ Never

We will never send an email on your behalf, delete a calendar event, or modify a file in your cloud storage. We read and display. That's it.

05Data Ownership

Your data belongs to you. We do not train AI models on your data. We do not sell your data. We do not share your data with third parties. You can export all of your data at any time in standard formats. If you delete your account, your data is purged.

06Production Security

The platform includes a production readiness check that verifies 10 security requirements before deployment:

AuthenticationSession-based auth with secure, HTTP-only cookies. Rate limiting on login attempts. Password hashing with bcrypt.
Column whitelistsEvery API endpoint that modifies data has explicit column whitelists preventing mass-assignment attacks.
CORS enforcementCross-origin requests are restricted to configured domains only.
Undo systemEvery destructive action has a 30-second undo window, storing reverse data for instant rollback.

07Incident Response

In the event of a security incident, we will notify affected users within 72 hours via email, provide a detailed incident report, and implement corrective measures. Security issues can be reported to security@myteam360.ai.
